# Agent API analyser

Короткий прогон для сбора чернового отчета по read-вызовам.

1. Убедись, что в `.env` есть `ERC3_API_KEY` и `OPENAI_API_KEY` (openai).
2. Из корня репозитория: `python -m agent_api_analyser.run --model gpt-4o-mini`
   - по умолчанию дергаются только read-запросы; для write добавь `--allow-writes`.
3. Черновик сохраняется в `agent_api_analyser/api-report/<date>.md`.

Что делает:
- строит каталог `Req_*` классов из `erc3` (read/write помечаются по имени);
- вытягивает фикстуры (первые id проекта/сотрудника/клиента/тайм-энтри) безопасными запросами;
- генерирует сценарии через OpenAI с контекстом из `docs/` и каталога;
- выполняет сценарии (write-операции опциональны) и пишет короткий отчет + идеи врапперов.
